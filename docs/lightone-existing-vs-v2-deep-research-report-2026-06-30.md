## 1. 결론

LIGHTONE의 기존 버전은 **카메라·조명·촬영 구도 품질을 확인하는 측정 신뢰도 검증 MVP**다. 즉, “AI 체형분석 서비스”라기보다 **분석 전에 입력 이미지가 비교 가능한 조건인지 확인하는 기술 증거**에 가깝다.

LIGHTONE_V2는 기존 기술 MVP를 버리지 않고, 이를 **비의료 PT 상담 리포트 SaaS의 신뢰도 보조 레이어**로 재배치한다. V2의 핵심은 카메라가 아니라, PT 현장의 회원 변화 기록, 통증 반응, 운동 수행, 생활습관 데이터를 구조화해 **재등록 상담과 트레이너 상담 품질 표준화**에 활용하는 것이다.

따라서 가장 현실적인 방향은 다음이다.

> 기존 버전 = 촬영 QC·캘리브레이션·조명정규화 기술 증거  
> V2 = 비의료 PT 상담 리포트 SaaS + 기능적 컨디셔닝 OS  
> 최종 포지셔닝 = 기술 MVP를 상담 리포트의 신뢰도 보조 장치로 통합한 B2B 웰니스 소프트웨어

## 3. 기존 버전의 본질

### 3.1 기존 버전의 핵심 정의

기존 버전은 “분석 모델”보다 **입력 신뢰도 검증 레이어**에 가깝다. 저장소의 포트폴리오 요약은 LIGHT ONE을 다음 질문에 답하는 프로젝트로 정의한다.

> Can the input image be trusted enough before running posture or body-reference analysis?

즉, 기존 버전의 핵심 가치는 다음이다.

1. 촬영 조건이 표준에 맞는지 확인한다.
2. 카메라 왜곡과 재투영 오차를 계산한다.
3. 조명 균일도와 그림자 비율을 수치화한다.
4. 피험자 구도, 배경, 바닥 기준 마커를 확인한다.
5. 품질을 PASS/CHECK/FAIL로 분류한다.

### 3.2 기존 버전의 구현 자산

| 모듈 | 구현 위치 | 기능 | 사업적 의미 |
|---|---|---|---|
| Camera Calibration | `camera_calibration/` | 체커보드 기반 보정, 내부 파라미터, 왜곡 계수, 재투영 오차 | 촬영 데이터의 기하학적 신뢰도 보조 |
| Lighting Normalization | `lighting_normalization/` | 화이트밸런스, CLAHE, 평균 밝기 정규화, 균일도·그림자 계산 | 조명 차이로 인한 전후 비교 왜곡 완화 |
| Center QC | `demo_assets/center_qc/` | 구도, 배경, 바닥 마커, 스케일 기준 확인 | 서로 다른 촬영자·환경 간 비교 가능성 보조 |
| Demo Pipeline | `demo_pipeline.py` | P1 카메라, P2 조명, P3 촬영 QC 통합 실행 | 심사용 기술 실행 증거 |
| Demo Result | `demo_result.json` | 합성 데이터 기반 PASS 결과와 수치 로그 | 코드 경로 검증 증거 |

### 3.3 기존 버전의 강점

- 기술 구현 범위가 명확하다.
- 실제 회원 이미지나 민감정보 없이 합성 데이터로 테스트한다.
- 의료 진단·치료를 하지 않는다는 안전 경계가 반복적으로 명시되어 있다.
- 측정 신뢰도 문제를 먼저 다루므로, 단순 “AI 체형분석”보다 기술적 설득력이 있다.
- 심사위원에게 “멘토 피드백을 실제 보정 모듈로 구현했다”는 증거로 제시하기 좋다.

### 3.4 기존 버전의 한계

| 한계 | 설명 | 대응 방향 |
|---|---|---|
| 고객가치가 간접적 | 촬영 QC만으로는 PT샵 원장이 돈을 낼 이유가 약함 | 상담 리포트 SaaS와 결합 |
| 실제 성능 검증 부족 | 현재 데모는 합성 데이터 기반 | 실제 센터 체커보드·그레이카드·마커 촬영 데이터 확보 |
| Web/API 미구현 | 포트폴리오 문서에 미구현 항목으로 명시됨 | Django/FastAPI 상담 입력폼부터 구현 |
| 체형분석 본체 부재 | 포즈추정·체형분석 모델은 아직 별도 실험 단계 | V2에서는 메인 상품으로 주장하지 않음 |
| 수익모델 연결 약함 | 기술은 있으나 구매자 문제와 직접 연결이 약함 | 재등록 상담, 상담 품질관리, 원장 대시보드로 연결 |

## 4. V2의 본질

### 4.1 V2의 핵심 정의

V2 문서에서 LIGHTONE_V2는 다음과 같이 정의된다.

> PT샵·헬스장·필라테스 센터가 회원의 운동 수행 기록, 통증 반응, 자세 변화, 생활습관 데이터를 구조화해 재등록 상담과 트레이너 상담 품질관리에 활용할 수 있도록 돕는 비의료 PT 상담 리포트 SaaS.

여기서 중요한 변화는 다음이다.

- AI 체형진단 → 비의료 PT 상담 리포트 SaaS
- 측정 정확도 경쟁 → 상담 품질 표준화
- D2C 앱 → B2B 센터/트레이너 도구
- 통증 원인 규명 → 통증 반응 기록과 트레이너 검토
- 기술 기능 → 사업 프로세스와 수익모델

### 4.2 V2의 핵심 사업 구조

| 항목 | V2 정의 |
|---|---|
| 구매자 | PT샵 원장, 헬스장 대표, 필라테스 센터 운영자 |
| 사용자 | PT 트레이너 |
| 수혜자 | PT 회원 |
| 1차 제품 | 재등록 상담 리포트 SaaS |
| 핵심 기술 | QS/JATC, 촬영 QC, 카메라 캘리브레이션, 조명 정규화, Basic/Review/Retake 리포트 |
| 차별점 | 상담 품질 표준화 + 기능적 컨디셔닝 방법론 IP |
| 규제 포지션 | 비의료 운동상담 보조 도구 |

### 4.3 V2의 강점

- 고객이 명확하다: PT샵 원장, 센터 운영자, 트레이너.
- 지불 이유가 명확하다: 재등록 상담, 변화 근거 제시, 상담 품질 관리.
- 기존 기술 MVP를 버리지 않고 “신뢰도 보조 레이어”로 재사용한다.
- 비의료 언어로 재정렬되어 규제 리스크가 줄었다.
- 모두의창업 관점에서 고객문제, 구현증거, 수익모델을 한 줄로 연결할 수 있다.

### 4.4 V2의 한계

| 한계 | 설명 | 대응 방향 |
|---|---|---|
| 실제 고객검증 부족 | 원장·트레이너 인터뷰와 가격 수용도 확인이 남아 있음 | 원장 5명, 트레이너 5명 인터뷰 실행 |
| SaaS 구현 부족 | 현재 저장소는 촬영 QC 중심. 상담 입력폼·리포트 생성기는 아직 미완성 | 최소 Django/FastAPI 입력폼 + 리포트 PDF부터 구현 |
| QS/JATC 검증 부족 | 지표 정의는 있으나 가중치·신뢰도 검증 전 | 전문가 기준 초기값 + 파일럿 데이터로 보정 |
| 수익모델 가설 단계 | 월 29,000/99,000/149,000원 가격은 검증 전 | 가격 테스트와 파일럿 LOI 필요 |
| 교육·인증은 후속 모델 | 방법론 IP는 강하지만 아직 커리큘럼·시험·인증체계 없음 | Level 1 커리큘럼 초안부터 작성 |

## 6. 기술 관점 심층 평가

### 6.1 현재 기술 완성도

현재 저장소의 기술 완성도는 “제품 완성”이 아니라 “기술 증거 MVP” 수준이다.

| 기술 | 상태 | 판단 |
|---|---|---|
| 카메라 캘리브레이션 | 구현됨 | 합성 데이터 기반 코드 경로 검증 완료. 실제 센터 데이터 필요 |
| 조명 정규화 | 구현됨 | 균일도·그림자 비율 계산 가능. 실제 조명 패치 데이터 필요 |
| Center QC | 구현됨 | 구도·배경·마커 확인 가능. 실제 촬영 SOP 수치 필요 |
| 통합 데모 | 구현됨 | P1/P2/P3를 하나로 실행하는 증거 있음 |
| 테스트 | 구현됨 | 합성 데이터 기준 통과. 실제 성능 증명은 아님 |
| 상담 입력폼 | 계획 단계 | Django/FastAPI 구현 필요 |
| 리포트 생성기 | 계획 단계 | PDF/웹 리포트 MVP 필요 |
| QS/JATC 엔진 | 문서화 단계 | 규칙 기반 계산식과 샘플 데이터 필요 |
| MediaPipe/OpenPose | 후속 실험 | V2 초기 핵심으로 두면 안 됨 |

### 6.2 기술 포지셔닝의 적정성

기존 버전이 가장 강한 영역은 “체형분석 결과”가 아니라 “분석 전 데이터 품질 확인”이다. V2에서 이 기술을 메인 상품으로 주장하면 피트릭스·엑스바디·바디닷 같은 장비/분석 기업과 정면충돌한다.

따라서 기술 포지셔닝은 다음처럼 유지해야 한다.

> LIGHTONE의 촬영 QC 기술은 의료적 판단이나 체형진단을 수행하지 않는다. 상담 리포트에 사용되는 촬영 입력값이 비교 가능한 조건인지 확인하고, 품질이 낮은 경우 트레이너 검토 또는 재촬영 대상으로 분류한다.

## 7. 사업 관점 심층 평가

### 7.1 기존 버전의 사업성

기존 버전 단독으로는 사업성이 제한적이다. PT샵 원장이 “카메라 보정 모듈” 자체에 비용을 지불할 가능성은 낮다. 반면 재등록 상담 리포트, 회원 변화 설명자료, 트레이너별 상담 품질 관리로 연결하면 구매 이유가 생긴다.

즉, 기존 기술은 **상품**이 아니라 **상품 신뢰도를 높이는 내부 엔진**으로 두는 것이 맞다.

### 7.2 V2의 사업성

V2는 고객과 과금 구조가 더 명확하다.

| 수익모델 | 우선순위 | 판단 |
|---|---:|---|
| 센터 월정액 SaaS | 1 | 가장 현실적. 원장/센터가 지불주체 |
| 트레이너 교육·인증 | 2 | 송광일 방법론 IP를 자산화하는 핵심 후속 모델 |
| 기업 B2B 근골격 예방 | 3 | 공모전·사회적 의미·반복매출 측면에서 강함. 단 영업난도 있음 |
| D2C 앱 | 4 | 리텐션·마케팅비·빅테크 경쟁 때문에 후순위 |
| 병원 협력 | 5 | 가능성은 있으나 규제·책임 구조가 복잡해 중장기 옵션 |

### 7.3 모두의창업 관점

V2가 기존 버전보다 심사 대응에 유리한 이유는 다음이다.

1. 고객문제가 더 구체적이다.
2. 기술 구현 증거가 이미 있다.
3. 규제 경계가 정리되어 있다.
4. 수익모델이 B2B SaaS로 설명 가능하다.
5. 멘토 피드백을 캘리브레이션·조명정규화 모듈로 반영했다는 서사가 가능하다.

## 10. 권장 아키텍처

```text

  ↓
운동 수행 기록 + RPE + 통증 반응 + 생활습관 + 촬영 기록

  ↓

  ↓

  ↓

  ↓

```

이 구조에서 AI 또는 자동화는 최종 판단자가 아니라 **입력 구조화와 리포트 초안 생성을 돕는 보조 역할**로 제한한다.

## 13. 최종 추천 전략

### 13.1 제품 전략

1. V2의 첫 제품은 “재등록 상담 리포트 생성기”로 고정한다.
2. 촬영 QC는 별도 판매 기능이 아니라 리포트 신뢰도 보조 기능으로 포함한다.
3. Basic/Review/Retake 리포트 샘플을 실제 데모처럼 작동하게 만든다.
4. QS/JATC는 처음부터 ML이 아니라 규칙 기반으로 시작한다.
5. MediaPipe/OpenPose는 후속 실험 브랜치로 분리한다.

### 13.2 사업 전략

1. PT샵 원장과 트레이너를 1차 고객으로 고정한다.
2. D2C 앱은 보류한다.
3. 교육·인증은 1차 고객검증 후 Level 1 커리큘럼으로 시작한다.
4. 기업 B2B 근골격 예방은 모두의창업 발표에서 확장성 카드로 사용한다.
5. 병원 협력은 규제 검토 전까지 중장기 옵션으로만 둔다.

### 13.3 심사 전략

심사 발표에서는 다음 구조가 가장 방어 가능하다.

```text
문제: PT 현장 데이터가 흩어져 재등록 상담과 상담 품질 관리가 어렵다.
해결: 운동 기록, 통증 반응, 자세 변화, 생활습관을 구조화해 비의료 상담 리포트를 생성한다.
기술: 촬영 QC, 캘리브레이션, 조명정규화로 리포트 입력값의 비교 가능성을 보조한다.
검증: 합성 데이터 기반 코드 경로 검증 완료, 실제 센터 파일럿 예정.
수익: 센터 월정액 SaaS → 트레이너 교육·인증 → 기업 B2B 확장.
안전: 진단·치료·처방·재활·질환위험도 표현을 배제한다.
```

## 15. 최종 판단

LIGHTONE의 기존 버전은 버릴 것이 아니라, V2의 기술적 신뢰도를 설명하는 핵심 근거다. 다만 기존 버전을 그대로 “AI 체형분석 솔루션”으로 팔면 경쟁과 규제 리스크가 커진다.

반대로 V2처럼 **비의료 PT 상담 리포트 SaaS**로 정리하면 기존 기술은 촬영 입력값의 비교 가능성을 높이는 차별적 보조 레이어가 된다. 이 방향은 GitHub 기술 증거, 모두의창업 심사 대응, 송광일의 현장 경험, 기능적 컨디셔닝 방법론 IP를 하나의 사업 서사로 묶을 수 있다.

따라서 다음 단계는 “더 큰 AI 모델”이 아니라, **더 작은 상담 리포트 MVP**다. 지금 필요한 것은 포즈추정 고도화가 아니라 고객이 보고 이해할 수 있는 Basic/Review/Retake 리포트, QS/JATC 계산표, 원장 인터뷰, 실제 촬영 SOP다.
