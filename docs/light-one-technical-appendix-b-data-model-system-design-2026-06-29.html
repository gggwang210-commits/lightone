LIGHT ONE — 비의료 PT 상담 리포트 SaaS

작성 기준일: 2026-06-29 · 기술검증 부록 / 시스템 설계 / 데이터 모델 설계

문서 성격 및 주의

본 부록은 의료 진단·치료·처방 목적이 아니라, PT 현장의 운동상담 데이터 표준화와 리포트 생성 가능성을 검토하기 위한 기술설계 자료다.

LIGHT ONE은 PT 현장에서 반복적으로 생성되는 운동 수행, 통증 반응, 자세 변화, 생활습관 데이터를 구조화하고, QS/JATC 지표를 기반으로 트레이너가 검토할 수 있는 비의료 운동상담 리포트를 생성하는 SaaS다.

4.1 설계 방향과 기본 원칙

1. PT 현장 데이터 우선 원칙

LIGHT ONE은 병원 EMR, 웨어러블, 대규모 임상 데이터보다 실제 PT 현장에서 매 세션 반복 생성되는 데이터를 우선한다.

  • 상담 기록
  • 운동 수행 기록
  • 세트·반복·중량
  • RPE
  • 통증 반응
  • 자세 변화
  • 생활습관 기록
  • 세션 후 피드백

2. 통증·자세·수행·생활습관의 단일 타임라인화

통증, 자세, 근기능, 생활습관을 별도 파일이나 분리된 앱으로 관리하지 않고 한 사람의 변화 과정을 동일한 시간축 위에 정렬한다.

3. 트레이너·회원·AI 3자 관점 통합

동일한 데이터를 트레이너에게는 운동상담 의사결정 근거로, 회원에게는 이해 가능한 상담 리포트로, AI에는 향후 패턴 분석과 리포트 자동화를 위한 구조화 데이터로 제공한다. 단, AI는 최종 판단자가 아니라 트레이너가 검토할 리포트 초안과 주의 신호를 생성하는 보조 도구다.

4. 단계적 AI 고도화 가능성

초기 MVP는 규칙 기반 평가와 간단한 통계 분석으로 시작하고, 데이터 축적 이후 머신러닝 기반 변화 패턴 분석, 세션 이탈 가능성 탐색, 주의 신호 분류 등으로 확장한다.

4.2 통합 데이터 레이어 구조

통합 데이터 모델은 세 레이어로 구성된다.

  1. 회원 정보 레이어
  2. 평가·세션 레이어
  3. 분석·지표 레이어

각 레이어는 독립된 테이블 또는 컬렉션으로 구성되며, Member_ID와 Session_ID를 기준으로 연결된다.

회원 정보 레이어

회원의 장기적 배경 정보를 저장한다. 초기 MVP에서는 필수 항목과 선택 항목을 구분하여 민감정보 수집 위험을 줄인다.

구분항목
필수Member_ID, 성별, 연령대, 주요 운동 목표, 운동 경험, 주요 불편감 부위, 세션 참여 빈도, 동의 상태
선택직업군, 활동 환경, 수면 시간, 좌식 시간, 스트레스 수준, 운동 제한 경험, 과거 부상 경험, 생활습관
고위험 민감진단명, 수술 이력, 약물 복용, 질환 정보, 의료기관 치료 이력

고위험 민감 항목은 의료적 판단 목적이 아니라 운동상담 시 주의가 필요한 참고정보로만 사용하며, 수집 시 별도 동의와 접근권한 관리가 필요하다.

평가·세션 레이어

평가·세션 레이어는 시간에 따라 반복적으로 생성되는 데이터를 저장하는 LIGHT ONE의 핵심 데이터 원천이다.

평가 데이터

  • 자세 평가: 정면/측면 사진, 주요 관절 정렬, 머리 전방 위치, 어깨 높이 차이, 골반 기울기, 무릎 정렬, 발 아치 상태, 좌우 비대칭
  • 기능 평가: 스쿼트, 런지, 힙힌지, 팔 거상, 한 발 서기, 호흡 패턴, 균형·안정성
  • 통증 반응 평가: 통증 부위, NRS, 발생 시점, 유발 동작, 완화 요인, 세션 중·후 변화
  • 근기능·운동성과 평가: 반복 가능 횟수, 수행 중단 여부, 세트 완료율, 좌우 수행 차이, RPE, 주관적 피로도

세션 데이터

  • 세션 메타: Session_ID, Member_ID, 날짜, 시간, 담당 트레이너, 세션 목적, 세션 상태
  • 운동 수행: 운동 이름, 세트, 반복, 중량, 휴식, 수행 완료 여부, RPE, 특이사항
  • 통증 반응: 운동별 통증 발생 여부, 통증 강도, 발생 세트, 즉시 조정 내용, 운동 중단 여부, REVIEW 필요 여부
  • 세션 후 피드백: 만족도, 주관적 피로도, 통증 변화 인식, 다음 세션 주의점, 회원·트레이너 코멘트

영상·이미지 데이터

영상 원본은 별도 파일 저장소에 저장하고, 분석 결과는 Media_ID, Session_ID, 운동 이름, 촬영 조건, 관절 좌표, 관절각, 비대칭 지표, 추정 신뢰도, 분석 실패 여부 같은 메타데이터로 저장한다.

분석·지표 레이어

회원 정보와 세션 데이터를 기반으로 생성되는 2차 지표를 저장한다. 이 레이어는 트레이너, 회원, AI가 같은 데이터를 서로 다른 방식으로 활용할 수 있게 하는 인터페이스다.

영역예시 지표
통증 반응부위별 NRS 변화, 세션 전후 NRS 변화, 운동별 통증 발생 빈도, REVIEW 필요 여부
자세머리 전방 위치, 어깨 높이 차이, 골반 기울기, 무릎 정렬, 체간 기울기, 좌우 비대칭, 추정 신뢰도
운동 수행반복 완료율, 세트 완료율, 휴식 준수율, RPE 변화, 세션 볼륨, QS
생활습관수면 시간, 좌식 시간, 스트레스 수준, 활동량, 생활습관 변화 메모
통합QS, JATC, 주의 신호 등급, 상담 우선순위, 리포트 생성 상태, 트레이너 검토 상태

분석·지표 레이어는 의료 진단을 위한 점수 체계가 아니라 비의료 운동상담 리포트와 트레이너 의사결정 보조를 위한 현장 지표다.

4.3 핵심 데이터 항목 정의

통증 반응 데이터

통증 데이터는 의료 진단 코드가 아니라 PT 현장에서 운동상담 참고자료로 활용 가능한 언어로 정의한다. 통증 강도는 NRS 0–10으로 기록한다. “통증 원인 판별”이 아니라 “통증 반응 기록”으로 표현한다.

자세 데이터

자세 데이터는 트레이너의 주관적 관찰을 보조하기 위해 정량화한다. 포즈 추정 결과는 절대적 진단값이 아니라 세션 간 변화 추적을 위한 참고값이다.

근기능·운동성과 데이터

근기능 데이터는 단순 근육량보다 실제 운동 수행 가능성에 초점을 둔다. 통증 증가, RPE 급상승, 반복 완료율 저하를 같은 세션 타임라인에서 확인한다.

생활습관 데이터

초기 MVP에서는 수면, 좌식 시간, 스트레스, 활동량, 수분 섭취, 식사 패턴 메모 등 최소 항목만 수집한다. 예측 모델의 핵심값보다 상담 리포트의 참고 정보로 사용한다.

4.4 시스템 아키텍처 설계

시스템 흐름은 데이터 수집 → 데이터 저장·정리 → 분석·지표 산출 → 피드백·리포트의 네 단계로 구성한다.

데이터 수집

회원 입력, 트레이너 입력, 영상 입력으로 구성한다. MVP에서는 스마트폰 1대와 표준 촬영 가이드를 기준으로 설계한다.

데이터 저장·정리

구조화 데이터와 비정형 데이터를 분리한다. 영상 원본은 별도 저장소에 저장하고, 분석 테이블에는 파일 경로, 촬영 조건, 분석 결과 메타데이터만 저장한다.

분석·AI 모델링

  1. 규칙 기반 평가: NRS 급증, 반복 완료율 저하, 휴식 준수율 저하, QS 저하에 따른 REVIEW 표시
  2. 통계·지표 분석: NRS, QS, JATC, 세션 이행률, 프로그램 준수율, RPE 추세, 세션 볼륨 변화 계산
  3. AI 기반 패턴 분석: 주의 신호 탐지, 세션 이탈 가능성 탐색, RPE 급증 패턴, 리포트 문장 생성, 상담 우선순위 제안

AI 결과는 항상 트레이너 검토를 거친다. 질병 예측, 통증 원인 판별, 치료 효과 예측, 자동 처방, 재활 효과 검증 표현은 사용하지 않는다.

피드백·리포트

최종 출력은 트레이너용 대시보드와 회원용 리포트다. 트레이너 대시보드는 오늘 세션에서 확인해야 할 내용을 보여주고, 회원 리포트는 변화 흐름을 직관적으로 전달한다.

4.10 소결

본 장은 PT 현장 데이터를 중심에 둔 통합 데이터 모델과 시스템 아키텍처를 제시한다. 병원 EMR이나 웨어러블 데이터가 아니라 PT 세션에서 실제로 생성되는 데이터를 1차 데이터로 설정하고, 통증 반응, 자세 변화, 운동 수행, 생활습관을 단일 타임라인으로 연결했다.

본 시스템은 의료 진단이나 치료를 목적으로 하지 않는다. LIGHT ONE은 트레이너가 검토할 수 있는 비의료 운동상담 리포트를 생성하고, 회원이 자신의 변화 과정을 이해할 수 있도록 돕는 PT 상담 리포트 SaaS다.

반영한 핵심 수정

  • 처방 → 운동 계획 조정
  • 통증 완화 → 통증 반응 기록
  • 체형 교정 → 자세 변화 추적
  • 재활 연계 → 운동상담 연계
