# LIGHT ONE 기술 구조 요약

## 1. 문서 목적

이 문서는 LIGHT ONE 저장소의 현재 구현 범위, 데이터 흐름, 미구현 범위, 안전 경계를 사업계획서와 포트폴리오에 바로 반영할 수 있도록 정리한 기술 구조 요약입니다.

LIGHT ONE은 현재 완성된 상용 서비스가 아니라, PT·피트니스 현장에서 상담 리포트의 신뢰도를 보조하기 위한 촬영 품질 확인, 조명 품질 측정, 카메라 보정 기술 증빙 MVP입니다.

## 2. 서비스 포지션

LIGHT ONE의 현재 포지션은 비의료 운동상담 보조 SaaS입니다.

- 구매자: PT샵 대표, 헬스장 대표, 센터 운영자
- 사용자: PT 트레이너
- 수혜자: PT 회원
- 핵심 가치: 회원 변화 기록을 정리하고, 상담 전 리포트 생성과 재등록 상담을 보조하는 것

본 프로젝트는 의료 진단, 치료 판단, 질병 예측, 치료 효과 평가, 자동 운동 처방을 제공하지 않습니다. 결과는 트레이너가 상담 과정에서 참고하는 보조 정보로만 사용해야 합니다.

## 3. 현재 구현 범위

| 구분 | 위치 | 현재 역할 |
|---|---|---|
| Camera Calibration | `camera_calibration/` | 체커보드 기반 카메라 보정, 재투영 오차 계산, PASS/CHECK/FAIL 분류 |
| Lighting Normalization | `lighting_normalization/` | 화이트밸런스, CLAHE, 평균 밝기 정규화, 조명 균일도와 그림자 비율 계산 |
| Center QC | `center_qc/`, `demo_assets/center_qc/` | 촬영 구도, 단색 배경, 바닥 기준 마커 확인 |
| Integrated Demo | `demo_pipeline.py` | 합성 데이터 기반 통합 데모 실행 및 `demo_result.json` 생성 |
| Quality Pipeline | `quality_pipeline.py` | 조명 패치, 그레이카드, 마커 프레임 대상 품질 리포트 생성 |
| Tests | `tests/` | 합성 데이터 기반 코드 경로와 판정 로직 검증 |

## 4. 데이터 흐름

```text
촬영 환경 준비
  ↓
카메라 캘리브레이션
  ↓
조명 정규화 및 균일도 검사
  ↓
촬영 부스 QC 게이트
  ↓
PASS / CHECK / FAIL 품질 상태 산출
  ↓
상담 리포트에 사용할 입력 데이터 신뢰도 보조
```

현재 저장소는 회원의 실제 신체 이미지나 민감 건강정보를 처리하는 운영 서비스가 아닙니다. 테스트와 데모는 합성 데이터 또는 비식별 샘플을 기준으로 동작합니다.

## 5. 실행 방법

의존성 설치:

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

테스트 실행:

```bash
python -m unittest discover -s tests
```

통합 데모 실행:

```bash
python demo_pipeline.py
```

정상 실행 시 카메라 캘리브레이션, 조명 정규화, 촬영 부스 QC 게이트 결과가 출력되고 `demo_result.json`이 생성됩니다.

## 6. 품질 판단 기준

| 영역 | 주요 지표 | 기준 |
|---|---|---|
| 카메라 보정 | 재투영 평균 오차 | 0.5px 미만 PASS, 0.5px 이상 1.0px 미만 CHECK, 1.0px 이상 FAIL |
| 조명 품질 | 밝기 균일도 | 90% 이상 PASS, 80% 이상 90% 미만 CHECK, 80% 미만 FAIL |
| 촬영 구도 | 중심 오차 | 프레임 폭의 15% 이하 권장 |
| 전신 포함 | 피험자 높이 | 프레임 높이의 45% 이상 권장 |
| 배경 통제 | 배경 색 이탈 | 기준값 12.0 이하 |
| 스케일 기준 | 바닥 기준 마커 | 검출 필수 |

위 기준은 현재 MVP 데모와 보완 면접 설명을 위한 기술 기준입니다. 실제 센터 성능 검증 수치로 주장하려면 실제 센터 촬영 데이터 기반 재검증이 필요합니다.

## 7. 미구현 범위

다음 항목은 현재 구현 완료로 표현하면 안 됩니다.

- 실제 웹 UI
- 로그인 및 계정 시스템
- 데이터베이스 저장
- 실제 회원 이미지 업로드 서비스
- 결제 기능
- 클라우드 배포
- 실제 MediaPipe 또는 OpenPose 연동
- 실제 체형 분석 AI 모델
- 자동 운동 처방
- 의료적 위험도 예측

## 8. 향후 개발 순서

1. 상담 입력폼 IA 설계
2. 규칙 기반 상담 리포트 생성기 구현
3. AUTO / REVIEW / BLOCK 라우팅 설계
4. 트레이너 검토 플로우 적용
5. 데모 상담 리포트 3종 작성
6. 개인정보 수집·이용 동의 플로우 초안 작성
7. 실제 센터 파일럿 데이터 수집 기준 정의
8. MediaPipe 또는 OpenPose 실험은 별도 브랜치에서 분리 진행

## 9. 안전 및 개인정보 경계

저장소에 올리면 안 되는 항목:

- 실제 회원 이미지
- 얼굴 이미지
- 민감 건강정보
- API Key
- `.env` 파일
- DB 접속정보
- 실명, 연락처, 회원번호 등 식별 가능 정보

사용하면 안 되는 표현:

- 진단
- 치료
- 재활
- 질환 위험도 평가
- 예후 예측
- 통증 위험도 예측
- 자동 운동 처방

상대적으로 안전한 표현:

- 운동상담 참고용
- 자세 체크
- 체형 관찰 리포트
- 상담 요약
- 촬영 품질 확인
- 측정 신뢰도 보조
- 트레이너 검토 필요 항목

## 10. 사업계획서 반영용 요약

LIGHT ONE은 PT 현장의 상담 기록과 촬영 데이터를 리포트화하기 전에 입력 이미지가 비교 가능한 조건인지 확인하는 품질 검증 레이어를 구현했습니다. 현재 MVP는 카메라 재투영 오차, 조명 균일도, 그림자 비율, 촬영 구도, 배경 통제, 바닥 기준 마커 검출 여부를 수치화하고 PASS/CHECK/FAIL로 분류합니다. 이를 통해 트레이너가 회원 상담 리포트를 만들 때 촬영 조건의 흔들림을 줄이고, 재촬영이 필요한 경우 그 사유를 확인할 수 있습니다.

단, 현재 데모 결과는 합성 데이터 기반 실행 증거이며 실제 센터의 정량 신뢰성 수치는 실제 설치 환경에서 체커보드, 그레이카드, 바닥 마커를 촬영한 뒤 별도로 산출해야 합니다.
