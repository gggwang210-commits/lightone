# LIGHT ONE 기술 프레임워크

## 데이터 기반 PT 의사결정 지원 시스템

> 이 문서는 LIGHT ONE의 핵심 기술 설계를 설명합니다.  
> 비의료 운동상담 보조 SaaS로서, 트레이너의 판단을 보조하는 Human-in-the-loop 구조입니다.

---

## 1. 핵심 지표 정의

### 1.1 QS — 수행 품질지수 (Quality Score)

**정의:** 운동 수행의 질적 수준을 세트 단위로 정량화하는 지표.  
운동량(볼륨)이 아닌 **수행 품질**을 관리한다.

#### 산출 요소

| 요소 | 설명 | 측정 방법 |
|---|---|---|
| 폼 정확도 (Form Accuracy) | 동작 자세의 정확성 | 체크리스트 / 영상 분석 |
| 반복수 달성률 (Rep Achievement) | 목표 대비 달성 비율 | 트레이너 기록 |
| 휴식 준수 (Rest Compliance) | 권장 휴식 시간 이행 여부 | 타이머 기록 |
| 통증 반응 (Pain Response) | 운동 중 통증 호소 여부 | 자기보고 (NRS) |

#### 등급화 방식

```text
충족 = 1.0점 / 부분충족 = 0.5점 / 미충족 = 0점
세트별 4항목 평균 → 세션 전체 QS (0~100점)
```

#### 계산 예시

```text
목표: 12회 / 실제: 10회 / 폼 정확도: 0.9 / 통증: 경미
QS = 100 × 0.9 × (10/12) - 5 = 69.2점
```

#### 임계값 및 라우팅

| QS 범위 | 등급 | 라우팅 | 처리 |
|---|---|---|---|
| ≥ 80점 | Good | AUTO | 점진 과부하 허용 (5~10% 증량) |
| 60~79점 | Fair | REVIEW | 트레이너 재확인 권고 |
| < 60점 | Poor | BLOCK | 부하 즉시 감량 또는 종목 교체 |

---

### 1.2 JATC — 통합 상태지수 (Joint Achievement & Tracking Composite)

**정의:** 통증·체형·근기능·생활습관을 하나의 숫자로 요약하는 세션별 종합 지표.  
"무엇이 병목인지"가 즉시 드러나도록 하위 인덱스를 함께 표시한다.

#### 구성요소 및 가중치 (초기 전문가 합의 기반)

| 구성요소 | 가중치 | 측정 변수 |
|---|---|---|
| 통증 (Pain) | 20% | NRS 0~10, 통증 부위 코드 |
| 체형·자세 (Posture) | 30% | 관절 각도, 비대칭 지수 |
| 근기능·훈련자극 (Function & Training) | 30% | ROM, FMS, 볼륨 로드, RPE |
| 생활습관 (Lifestyle) | 20% | 수면, 스트레스, 활동량 |

#### 산출 방식

```text
각 요소 → Z-score 또는 Min-Max 정규화
→ 가중 합산 → JATC 0~100점
```

#### 계산 예시

```text
통증 80점, 체형 70점, 근기능 75점, 생활습관 90점
JATC = (0.2×80) + (0.3×70) + (0.3×75) + (0.2×90) = 77.5점
```

#### 임계값 및 처리 규칙

| 조건 | 처리 |
|---|---|
| 통증 NRS ≥ 7 | 통증 우선 프로토콜 (세션 중단 + 전문가 상담 권고) |
| 비대칭 지수 > 15% | 교정 프로토콜 (가동성 훈련 우선) |
| JATC 총점 < 40 | 회복 프로토콜 (강도 축소, 수면·스트레스 점검) |

#### 병목 탐지 기능

JATC 총점이 높아도 하위 지수 하나가 낮으면 Red Flag 표시.  
예: 총점 75점이지만 자세 지수 40점 → 자세 교정 우선 중재.

#### 시각화

- 레이더 차트 (4개 하위 영역 동시 표시)
- 세션별 시계열 그래프 (변화 추세 추적)

## 2. 3계층 통합 데이터 모델

```text
Layer 1: 개인 프로필(Profile Layer)
- 회원ID, 신체정보, 건강력, 운동목표
- 관계: 회원 1 : 세션 N

Layer 2: 세션·세트 수행(Session/Set Layer)
- 세션ID, 운동종목, 세트, 반복, 중량
- QS, 통증반응, RPE, 타임스탬프
- 관계: 세션 1 : 세트 N

Layer 3: 상태·결과(Outcome Layer)
- 세션별 QS, JATC, 위험신호, 생활습관 변화
- 관계: 세션 1 : 상태지수 1
```

### 핵심 연결 키

```python
member_id   # 회원 식별자 (UUID)
session_id  # 세션 식별자 (UUID + timestamp)
```

### 최소 수집 변수 (MVP 기준)

| 도메인 | 변수 | 타입 | 수집 방법 |
|---|---|---|---|
| 통증 | NRS 0~10 | Numeric | 자기보고 (앱) |
| 통증 | 부위 코드 | Categorical | 체크리스트 |
| 체형·자세 | 관절 각도 | Numeric | 카메라 + 포즈 AI |
| 체형·자세 | 비대칭 지수 | Numeric | 산출값 |
| 근기능 | ROM (도) | Numeric | 측정 |
| 근기능 | FMS 점수 | Ordinal | 트레이너 평가 |
| 훈련 | 볼륨 로드 (kg) | Numeric | 트레이너 기록 |
| 훈련 | RPE (1~10) | Numeric | 자기보고 |
| 수행 | QS | Numeric | 체크리스트 산출 |

## 3. 위험 라우팅 구조 (AUTO / REVIEW / BLOCK)

```text
입력 데이터
  → 규칙 기반 평가 엔진
  → AUTO: QS ≥ 80 AND JATC ≥ 60 AND 통증 NRS ≤ 3
  → REVIEW: QS 60~79 OR JATC 40~59 OR 통증 NRS 4~6
  → BLOCK: QS < 60 OR 통증 NRS ≥ 7 OR JATC < 40
```

> ⚠️ BLOCK은 의료 진단이 아닙니다.  
> 안전 신호로 표시하며, 최종 판단과 상담 책임은 담당 트레이너에게 있습니다.

## 4. AI 고도화 로드맵

### Phase 1 — 규칙 기반 MVP (현재)

```text
구현 완료:
- camera_calibration/: 촬영 품질 PASS/CHECK/FAIL 분류
- lighting_normalization/: 조명 조건 자동 보정
- center_qc/: 촬영 환경 품질 게이트

개발 중:
- Django 기반 상담 입력폼
- 규칙 기반 QS·JATC 산출 엔진
- 트레이너 검토 플로우 (Human-in-the-loop)
- 상담 리포트 자동 생성
```

### Phase 2 — ML 기반 패턴 분석 (3~6개월)

```text
예정 기술 스택:
- XGBoost / LightGBM: 상담 패턴 분석 및 JATC 가중치 최적화
- MediaPipe / OpenPose v1.7.0: 자세 포인트 인식 연동
- 관절 가동범위(ROM), 비대칭 지수 자동 산출
- 표준 촬영 조건: 거리 2.5m, 조명 500lux 이상 [확인필요]
```

### Phase 3 — 개인화 AI 추천 (6~12개월)

```text
- 회원별 통증·체형·훈련 패턴 예측 모델
- QS 추세 기반 다음 세션 부하 자동 제안
- JATC 변화 기반 개인화 운동 처방 보조
- 연합학습(Federated Learning) 기반 프라이버시 보존 고도화
```

## 5. 비의료 안전 경계

| 제공하는 것 | 제공하지 않는 것 |
|---|---|
| 트레이너 입력 기반 상담 리포트 자동 생성 | 의료 진단 |
| QS·JATC 기반 주의 포인트 알림 | 질병 예측 |
| 운동 수행 기록 구조화 | 치료 효과 판단 |
| 재등록 상담용 변화 기록 | 자동 운동 처방 |
| 안전 신호 (비의료) | 임상 의사결정 지원 |

**표준 고지 문구:**

> 본 리포트는 운동 상담 및 체형 변화 기록을 위한 참고 자료이며, 의학적 진단이나 치료를 대체하지 않습니다. 최종 판단과 상담 책임은 담당 트레이너에게 있습니다.
