## 0. 핵심 요약

이 문서는 LIGHT ONE 핵심자료 DB에서 바로 제출·발표·심사 대응에 사용할 수 있는 내용을 3개 용도로 재구성한 실행형 자료다.

1. **사업계획서 기술 섹션**: QS·JATC·3계층 데이터 모델을 기술 구현 논리로 삽입한다.
2. **모두의창업 2기 신청서**: 건양대 바이오메디컬 AI 교육 과정 수강, OpenPose 근거 논문, Human-in-the-loop 구조를 실현가능성 근거로 쓴다.
3. **심사 질문 대응**: “AI가 정확히 무엇을 하느냐”는 질문에 규칙 기반 MVP → 포즈 AI 연동 → ML 개인화 고도화의 단계형 답변으로 대응한다.

## 1. 사업계획서 기술 섹션

### 삽입 위치

| 위치 | 삽입 내용 | 목적 |
|---|---|---|
| 제품·서비스 개요 | QS·JATC 정의 | 단순 운동 기록 앱이 아니라 의사결정 보조 SaaS임을 설명 |
| 기술 구현 현황 | 3계층 데이터 모델 | 실제 DB·MVP 구현 가능성 설명 |
| 추진 일정 | 규칙 기반 → ML → MediaPipe/OpenPose 로드맵 | 단계별 개발 현실성 제시 |
| 리스크 및 대응 | 비의료 고지, HITL, 민감정보 처리 | 의료기기 오인·개인정보 리스크 완화 |

### 바로 삽입 가능한 기술 설명 문단

LIGHT ONE은 PT샵·헬스장·필라테스 센터의 상담 품질을 표준화하기 위한 비의료 운동상담 보조 SaaS이다. 핵심 기술은 운동 수행 품질을 정량화하는 QS(Quality Score, 수행 품질지수)와 통증·체형·근기능·생활습관을 통합적으로 요약하는 JATC(Joint Achievement & Tracking Composite, 통합 상태지수)이다. QS는 폼 정확도, 반복수 달성률, 휴식 준수 여부, 통증 반응을 기준으로 세트 단위 수행 품질을 산출하며, JATC는 통증·체형·근기능·생활습관 데이터를 정규화해 회원 상태의 병목 요인을 표시한다.

MVP 단계에서는 트레이너 입력값과 체크리스트를 기반으로 QS와 JATC를 규칙 기반으로 산출한다. 이후 2단계에서는 MediaPipe 또는 OpenPose 기반 자세 포인트 인식을 연동해 관절 가동범위(ROM), 자세 정렬, 비대칭 지수 산출을 자동화한다. 3단계에서는 축적된 상담·운동·통증 데이터를 기반으로 XGBoost, LightGBM 등 머신러닝 모델을 적용해 개인별 변화 패턴과 다음 상담 우선순위를 제안하는 구조로 고도화한다.

LIGHT ONE의 AI는 트레이너를 대체하지 않는다. AI는 상담 리포트 초안 생성, 위험 신호 표시, 변화 추세 요약을 담당하고, 최종 판단과 상담 책임은 담당 트레이너가 수행하는 Human-in-the-loop 구조를 따른다.

## 2. 모두의창업 2기 신청서

### 신청서용 문단

창업자는 약 20년간의 퍼스널 트레이닝 및 헬스장 운영 경험을 통해 PT 현장에서 반복되는 문제를 직접 경험했다. 트레이너마다 상담 방식과 기록 품질이 다르고, 회원 재등록 상담 시 객관적인 변화 자료를 제시하기 어렵다는 문제가 있다. LIGHT ONE은 이러한 현장 문제를 해결하기 위해 PT 상담 데이터를 구조화하고, QS와 JATC 지표를 기반으로 회원 상태와 운동 수행 품질을 리포트화하는 비의료 운동상담 보조 SaaS이다.

현재 창업자는 건양대 바이오메디컬 AI 교육 과정과 KDT 기반 AI 학습을 통해 Python, Django, 머신러닝, 헬스케어 데이터 분석 역량을 강화하고 있다. 기술적으로는 1단계에서 규칙 기반 체크리스트와 상담 입력폼으로 MVP를 구현하고, 2단계에서 OpenPose 또는 MediaPipe 기반 자세 포인트 인식을 연동하며, 3단계에서 ML 기반 개인화 분석으로 확장할 계획이다. OpenPose 관련 선행연구(Cao et al., 2017)는 마커 없는 관절 포인트 추출의 기술적 근거로 활용 가능하다.

### 신청서에서 피해야 할 표현

| 피해야 할 표현 | 대체 표현 |
|---|---|
| 질병을 진단한다 | 운동 상담 참고 정보를 제공한다 |
| 통증 원인을 판별한다 | 통증 반응을 기록하고 주의 신호를 표시한다 |
| 치료 효과를 예측한다 | 운동 수행 변화와 상담 이력을 시각화한다 |
| 자동 처방한다 | 트레이너의 의사결정을 보조한다 |
| AI가 회원을 관리한다 | 트레이너가 검토하는 리포트 초안을 생성한다 |

## 3. 심사 질문 대응

### “AI가 정확히 뭘 하나요?” 30초 답변

현재 LIGHT ONE의 AI는 진단이나 처방을 하지 않습니다. MVP 단계에서는 트레이너가 입력한 상담·운동·통증 데이터를 기준으로 QS와 JATC를 규칙 기반으로 산출하고, 회원에게 보여줄 상담 리포트 초안을 자동 생성합니다. 2단계에서는 OpenPose 또는 MediaPipe를 연동해 자세 포인트와 비대칭 지수를 자동 산출하고, 3단계에서는 ML 기반으로 개인별 변화 패턴과 다음 상담 우선순위를 제안하는 구조로 고도화합니다.

### “AI가 정확히 뭘 하나요?” 90초 답변

LIGHT ONE에서 AI의 역할은 세 가지입니다. 첫째, 트레이너가 입력한 운동 수행 기록, 통증 반응, 생활습관 데이터를 구조화합니다. 둘째, QS와 JATC라는 지표로 운동 수행 품질과 회원 상태를 요약해 상담 리포트 초안을 만듭니다. 셋째, 위험 신호가 있으면 AUTO, REVIEW, BLOCK으로 분류해 트레이너가 다시 확인하도록 돕습니다. 현재 단계는 규칙 기반 MVP이고, 향후 MediaPipe/OpenPose를 연동해 관절 포인트, ROM, 비대칭 지수를 자동 산출할 계획입니다. 다만 최종 판단은 AI가 아니라 트레이너가 수행하며, 서비스는 의료 진단·치료·처방이 아닌 비의료 운동상담 보조 도구로 설계합니다.

### 예상 후속 질문 대응

| 질문 | 답변 방향 |
|---|---|
| 의료기기 아닌가요? | 진단·치료·처방 목적이 아니라 운동상담 보조와 기록 리포트 목적이다. 비의료 고지와 금지 표현 가이드를 적용한다. |
| 정확도는 어떻게 검증하나요? | 초기에는 규칙 기반 체크리스트 일관성, 전문가 평가와의 상관, 파일럿 ICC 검증을 계획한다. 포즈 AI는 촬영 조건을 표준화한 뒤 별도 검증한다. |
| 고객이 돈을 낼 이유는? | 원장은 재등록 상담과 트레이너 상담 품질 관리가 필요하다. LIGHT ONE은 인바디·눈바디 외에 상담 리포트라는 재등록 근거를 만든다. |
| 기존 CRM과 차이가 뭔가요? | CRM은 예약·결제·회원관리 중심이고, LIGHT ONE은 상담 리포트 자동화와 QS/JATC 기반 상담 품질 관리가 핵심이다. |
| 개인정보는 어떻게 처리하나요? | 최소 수집, 별도 동의, 목적 제한, 비식별 ID, 접근권한 관리, 감사 로그, HITL 구조를 적용한다. |

## 4. 기술 구조 요약

| 구성요소 | 현재 단계 | 설명 | 검증 필요 항목 |
|---|---|---|---|
| QS | MVP | 폼·반복·휴식·통증 기반 수행 품질 산출 | 현장 체크리스트 일관성 |
| JATC | MVP | 통증·체형·근기능·생활습관 통합 상태지수 | 가중치 타당도, 전문가 검증 |
| 3계층 데이터 모델 | MVP | Profile / Session-Set / Outcome Layer | DB 스키마와 실제 입력폼 연결 |
| MediaPipe/OpenPose | 2단계 | 자세 포인트 기반 ROM·비대칭 산출 | 촬영 조건, 정확도, 개인정보 보호 |
| ML 개인화 | 3단계 | QS·JATC 추세 기반 개인화 제안 | 데이터 수, 편향, 설명가능성 |

## 6. 바로 사용할 최종 문장

LIGHT ONE은 PT샵·헬스장·필라테스 원장이 트레이너 상담 품질을 관리하고 회원 재등록률을 높이기 위해 사용하는 월정액 PT 상담 리포트 SaaS입니다. QS와 JATC 지표를 기반으로 운동 수행 품질과 회원 상태를 구조화하고, 트레이너가 검토할 수 있는 상담 리포트 초안을 자동 생성합니다. 현재는 규칙 기반 MVP로 시작하며, 향후 MediaPipe/OpenPose 기반 자세 포인트 인식과 ML 기반 개인화 분석으로 고도화할 계획입니다. 본 서비스는 의료 진단·치료·처방이 아닌 비의료 운동상담 보조 서비스입니다.
