# LIGHT ONE 사업계획서용 기술 근거 정리

## 1. 문서 목적

이 문서는 LIGHT ONE 프로젝트를 사업계획서, 포트폴리오, 멘토링, 심사 발표에 활용하기 위해 현재 구현된 기술 근거와 미구현 범위를 구분해 정리한 자료입니다.

LIGHT ONE은 PT·피트니스 현장에서 회원 상담 리포트의 신뢰도를 높이기 위한 비의료 운동상담 보조 SaaS를 목표로 합니다. 현재 저장소는 완성된 상용 서비스가 아니라, 촬영 품질 확인, 조명 품질 측정, 카메라 보정, 촬영 부스 QC를 통해 입력 이미지가 비교 가능한 조건인지 확인하는 기술 증빙 MVP입니다.

## 2. 한 줄 요약

LIGHT ONE은 카메라, 조명, 촬영 구도 차이로 인해 상담 리포트의 전후 비교 신뢰도가 흔들리는 문제를 줄이기 위해 촬영 품질을 수치화하고 PASS/CHECK/FAIL로 분류하는 품질 검증 레이어입니다.

## 3. 현재 구현된 기능

| 구분 | 구현 위치 | 구현 내용 | 사업계획서 반영 의미 |
|---|---|---|---|
| 카메라 캘리브레이션 | `camera_calibration/` | 체커보드 기반 카메라 보정, 재투영 오차 계산, PASS/CHECK/FAIL 분류 | 촬영 장비 차이와 렌즈 왜곡을 수치로 관리할 수 있음 |
| 조명 정규화 | `lighting_normalization/` | 화이트밸런스, CLAHE, 평균 밝기 정규화, 조명 균일도와 그림자 비율 계산 | 조명 편차를 눈대중이 아니라 숫자로 확인할 수 있음 |
| 촬영 부스 QC | `center_qc/`, `demo_assets/center_qc/` | 피험자 구도, 단색 배경, 바닥 기준 마커 확인 | 촬영 전 재촬영 필요 여부를 자동 판단할 수 있음 |
| 통합 데모 | `demo_pipeline.py` | 합성 데이터 기반 카메라·조명·QC 통합 실행 | 심사자에게 실행 가능한 기술 증거를 보여줄 수 있음 |
| 테스트 | `tests/` | 합성 데이터 기반 20개 테스트 통과 | 코드가 재현 가능하게 동작함을 확인할 수 있음 |
