# LIGHT ONE 촬영 환경 편차 정량 보정 방안

대전창조경제혁신센터 보완 면접용  
2026.06.28

본 자료는 LIGHT ONE 체형분석 시스템의 촬영 환경 편차를 어떻게 줄이고, 그 결과를 어떤 숫자로 검증할지 설명하기 위한 보완 면접 자료입니다. LIGHT ONE은 체형 참고 정보를 제공하는 서비스이며, 의료적 진단을 대체하지 않습니다.

## 1. 문제 정의

### 심사 의견 직접 인용

> [확인필요: 실제 심사 의견 원문 삽입]

현재 보완해야 할 핵심은 “촬영할 때마다 카메라, 조명, 촬영 환경이 조금씩 달라지면 측정값도 흔들릴 수 있다”는 점입니다. 즉, 알고리즘의 정확도만 설명하는 것이 아니라, 입력 이미지가 매번 같은 조건으로 들어오도록 통제하고 그 통제 수준을 숫자로 증명해야 합니다.

촬영 편차는 크게 세 가지입니다.

- 카메라 편차: 렌즈 왜곡, 카메라별 내부 파라미터 차이, 4방향 카메라 간 차이
- 조명 편차: 센터 조명의 색온도, 밝기, 그림자, 하이라이트 차이
- 환경 편차: 피험자와 카메라 거리, 피험자 위치, 전신 포함 여부, 배경, 바닥 기준 마커 유무

## 2. 3가지 보정 방안

## 2.1 카메라 편차 보정

### 편차 원인

Canon EOS R7과 RF 35mm F1.8 렌즈를 고정 촬영에 사용하더라도 렌즈에는 미세한 왜곡이 있습니다. 이 왜곡이 보정되지 않으면 실제 직선이 이미지에서 휘어 보이고, 자세 각도 계산에 영향을 줄 수 있습니다. 또한 전/후/좌/우 4대 카메라는 각자 위치와 렌즈 상태가 다르기 때문에 독립적으로 보정해야 합니다.

### 보정 방법

9x6 체커보드를 촬영하여 각 카메라의 내재 파라미터 `K`와 왜곡계수 `D`를 계산합니다. OpenCV의 `cv2.calibrateCamera`를 사용하며, 4대 카메라를 각각 따로 캘리브레이션합니다.

계산된 `K`, `D`, `image_size`, `camera_id`는 JSON으로 저장합니다. 이후 실제 촬영 이미지는 저장된 보정값으로 왜곡 보정을 거친 뒤 측정에 사용합니다.

### 판정 기준

- 재투영 평균 오차 **0.5px 미만: PASS**
- 재투영 평균 오차 **0.5px 이상 1.0px 미만: CHECK**
- 재투영 평균 오차 **1.0px 이상: FAIL**

재투영 오차는 “체커보드의 실제 기준점이 보정된 카메라 모델에서 얼마나 같은 위치로 다시 찍히는가”를 픽셀 단위로 확인하는 값입니다. 숫자가 작을수록 카메라 모델이 실제 촬영 상태를 잘 설명합니다.

## 2.2 조명 편차 보정

### 편차 원인

센터 조명은 시간대, 조명 색온도, 주변 밝기, 피험자의 위치에 따라 달라질 수 있습니다. 같은 자세라도 그림자가 강하거나 색이 치우치면 마커나 윤곽 검출 결과가 달라질 수 있습니다.

### 보정 방법

조명 패치 또는 그레이카드 기준 이미지를 사용하여 다음 순서로 정규화합니다.

1. Gray-world 화이트밸런스로 색 편향을 줄입니다.
2. CLAHE로 국소 대비를 보정하여 그림자와 하이라이트 영향을 줄입니다.
3. 목표 평균 밝기에 맞추어 노출을 정규화합니다.

정규화 후에는 프레임을 격자로 나누고 각 격자의 밝기를 비교합니다. 가장 어두운 격자와 가장 밝은 격자의 차이가 작을수록 조명이 균일하다고 판단합니다.

### 판정 기준

- 밝기 균일도 **90% 이상: PASS**
- 밝기 균일도 **80% 이상 90% 미만: CHECK**
- 밝기 균일도 **80% 미만: FAIL 및 재촬영 권고**

균일도 계산식은 다음과 같습니다.

```text
균일도(%) = 100 * 가장 어두운 격자 평균 밝기 / 가장 밝은 격자 평균 밝기
```

그림자 영역 비율도 함께 계산합니다. 그림자가 기준 이상이면 측정값이 흔들릴 수 있으므로 재촬영을 권고합니다.

## 2.3 촬영 환경 편차 보정

### 편차 원인

피험자와 카메라의 거리, 피험자의 중심 위치, 전신 포함 여부, 배경 상태, 바닥 기준 마커가 회차마다 달라지면 같은 사람의 촬영 결과도 비교하기 어렵습니다. 특히 거리와 스케일이 달라지면 픽셀 단위 측정값을 실제 단위로 비교할 수 없습니다.

### 보정 방법

촬영 전에 자동 QC 게이트를 통과해야 측정을 진행합니다. QC 게이트는 얼굴이나 신원 정보를 저장하지 않고, 구도/배경/마커만 확인합니다.

검사 항목은 다음과 같습니다.

- 피험자가 프레임 중앙에 있는지
- 전신이 프레임 안에 충분히 포함되는지
- 배경이 단색으로 통제되어 있는지
- 거리/스케일 보정용 바닥 기준 마커가 보이는지
- 바닥 마커의 실제 길이를 기준으로 `px→cm` 스케일을 환산할 수 있는지

### 판정 기준

- 피험자 중심 오차 **프레임 폭의 15% 이하**
- 피험자 높이 **프레임 높이의 45% 이상**
- 배경 색 이탈 기준 **12.0 이하**
- 바닥 기준 마커 **검출 필수**

하나라도 실패하면 측정을 진행하지 않고 “다시 촬영하세요”와 실패 사유를 한국어로 반환합니다.

## 3. 구현 설계 현황

현재 Codex 산출물은 `outputs/` 아래에 구성되어 있습니다.

```text
outputs/
  camera_calibration/
    checkerboard_calibrate.py
    reprojection_error.py
    undistort.py
  lighting_normalization/
    normalize.py
    uniformity_check.py
    before_after_report.py
  center_qc/
    booth_spec.py
    qc_gate.py
    scale_reference.py
  demo_pipeline.py
  demo_result.json
  INTERVIEW_SUMMARY.md
```

### 카메라 캘리브레이션 모듈

- 체커보드 이미지로 카메라별 `K`, `D` 계산
- 재투영 오차 평균/표준편차/최댓값 계산
- 저장된 보정값으로 이미지 왜곡 보정
- 합성 데이터 기반 단위 테스트 포함

### 조명 정규화 모듈

- 화이트밸런스 보정
- CLAHE 기반 국소 대비 정규화
- 목표 평균 밝기 기반 노출 정규화
- 균일도와 그림자 비율 계산
- 전/후 히스토그램 및 품질 리포트 PNG 생성

### 촬영 부스 QC 모듈

- 촬영 부스 표준 스펙을 코드 상수와 README로 정의
- 아직 실측하지 않은 물리 스펙은 모두 `[확인필요]`로 표시
- 피험자 구도, 단색 배경, 바닥 마커 자동 검사
- 바닥 마커 실제 길이 기반 `px→cm` 스케일 환산

### 통합 데모 파이프라인

`outputs/demo_pipeline.py`는 실제 이미지 없이도 합성 데이터로 전체 흐름을 실행합니다.

```bash
python demo_pipeline.py
```

출력 항목은 다음과 같습니다.

- 카메라별 재투영 오차 평균 px 및 PASS/CHECK/FAIL
- 조명 균일도와 그림자 비율
- QC 게이트 항목별 PASS/FAIL 및 한국어 사유
- `outputs/demo_result.json` 저장

단, 데모의 합성 데이터 결과는 코드 동작 검증용입니다. 실제 센터 신뢰성 증거는 센터 설치 후 실제 체커보드, 조명 패치, 바닥 마커 촬영 데이터로 다시 산출해야 합니다.

## 4. 향후 검증 계획

### 1단계: 실제 센터 물리 스펙 확정

현재 `[확인필요]`로 둔 항목을 실측합니다.

- 카메라 높이
- 피험자-카메라 거리
- 전/후/좌/우 카메라 배치각
- 배경색과 배경 재질
- 바닥 마커의 실제 길이와 위치
- 허용 중심 오차와 각도 오차

임의값은 넣지 않고, 실제 설치 환경에서 측정한 값만 표준 스펙에 반영합니다.

### 2단계: 카메라별 캘리브레이션 데이터 확보

4대 카메라 각각에 대해 체커보드를 여러 위치와 각도에서 촬영합니다. 각 카메라별 JSON 보정값을 만들고, 재투영 평균 오차가 **0.5px 미만**인지 확인합니다.

### 3단계: 조명 패치 반복 촬영 검증

서로 다른 시간대와 조명 조건에서 조명 패치 또는 그레이카드를 촬영합니다. 정규화 전/후 균일도와 그림자 비율을 비교하고, 정규화 후 균일도가 **90% 이상**으로 유지되는지 확인합니다.

### 4단계: 운영자별 QC 통과율 검증

여러 운영자가 같은 SOP로 촬영했을 때 QC 게이트 통과율과 실패 사유를 기록합니다. 실패 사유가 반복되는 항목은 SOP 문구, 바닥 마커 위치, 카메라 고정 장치를 개선합니다.

### 5단계: 반복 측정 안정성 평가

동일 조건에서 여러 번 촬영하여 자세 각도의 반복성을 확인합니다. 이 단계에서는 카메라 보정, 조명 정규화, QC 게이트를 모두 통과한 데이터만 비교 대상으로 사용합니다.

## 5. 한계 및 전제 조건

- 본 시스템은 체형 참고 정보를 제공하며 의료적 진단을 대체하지 않습니다.
- 카메라 보정값은 카메라 위치, 렌즈, 초점, 해상도가 바뀌면 다시 계산해야 합니다.
- 조명 정규화는 과도한 그림자, 반사, 역광을 완전히 복구하지 못할 수 있습니다. 기준 미달이면 재촬영해야 합니다.
- 촬영 부스의 실제 높이, 거리, 각도, 배경색, 바닥 마커 길이는 반드시 실측 후 확정해야 하며, 현재 미확정 항목은 `[확인필요]`로 유지합니다.
- QC 게이트는 얼굴이나 신원 정보를 판단하지 않습니다. 구도, 배경, 마커처럼 측정 재현성과 관련된 조건만 검사합니다.
- 합성 데이터 데모는 구현 동작을 보여주는 용도이며, 대외 제출용 정량 신뢰성 수치는 실제 센터 촬영 데이터로 산출해야 합니다.
