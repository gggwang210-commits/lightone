## 0. 최종 결론

현재 3개 파일 중 외부 제출 최종 후보는 `LIGHTONE_인포그래픽_Rev6_시장경쟁비즈니스모델`입니다. 다만 Rev6는 시장·정책·경쟁사 수치가 많아 근거 검증 부담이 가장 큽니다. 현재 상태 그대로 제출하기보다는, 수치성 주장을 `확정 사실`이 아니라 `리서치 기반 가설` 또는 `[확인필요]`로 낮추는 수정이 필요합니다.

`lightone_report_member_v2`는 가장 안전합니다. SAMPLE DATA와 트레이너 검토 구조가 명확하므로 화면 설계 시안으로 사용 가능합니다. 단, QS·JATC 산식은 검증된 의학 점수가 아니라 내부 참고 지표임을 추가해야 합니다.

`lightone_infographic_v4`는 시스템 흐름도·설명자료로 적합합니다. 다만 “경쟁 솔루션이 멈추는 지점”처럼 경쟁사 기능을 단정하는 문장은 완화해야 합니다.

## A. `lightone_report_member_v2 (1).html`

### 현재 용도

회원 리포트 화면 설계 시안입니다. 세션 기록, 운동 로그, 통증/RPE, 자세 관찰, 촬영 QC, QS, JATC, 트레이너 검토 흐름을 보여주는 화면입니다.

### 종합 판정

사용 가능. 다만 `QS`, `JATC`, `주의 신호`, `PDF 다운로드` 관련 문구를 조금 더 안전하게 정리해야 합니다.

### 유지할 부분

| 항목 | 이유 |
|---|---|
| `SAMPLE DATA · 견본 화면` 표시 | 실제 회원 데이터가 아님을 명확히 하므로 안전함 |
| 입력 데이터 → 지표 계산 → 트레이너 검토 구조 | 자동 판단이 아니라 트레이너 검토를 거치는 구조라 리스크가 낮음 |
| 통증 전후, RPE, 자세 관찰, 촬영 QC 구분 | PT 현장에서 기록할 수 있는 항목으로 설계 방향이 적절함 |
| `주의 신호` 표현 | 의료 진단이 아니라 주의할 관찰값으로 보이면 사용 가능 |

### 수정해야 할 사항

| 우선순위 | 현재 표현/영역 | 문제점 | 수정 권장안 |
|---:|---|---|---|
| 1 | `QS (Quality Score)` / “세션 수행 품질을 0–100 스케일로 산출” | 산식 검증 전에는 객관적 품질점수처럼 보일 수 있음 | “내부 기준 기반 세션 참고 점수입니다. 실제 적용 전 파일럿 검증이 필요합니다.” 추가 |
| 1 | `JATC (통합 상태 추적)` | 약어 의미와 산식 근거가 불명확함 | 약어 풀네임 또는 “통합 상태 추적 지표(내부 가칭)”로 수정 |
| 1 | `통증 증가 추세 확인` | 의료적 위험 판단처럼 보일 수 있음 | “회원이 보고한 통증 점수 증가가 관찰됨”으로 수정 |
| 2 | `자세 관찰 신뢰도 낮음 (0.78)` | 0.78 기준이 무엇인지 불명확함 | “촬영·관찰 신뢰도 0.78: 트레이너 재확인 필요”로 수정 |
| 2 | `PDF 다운로드` | 실제 기능이 구현되지 않은 시안이면 오해 가능 | “PDF 다운로드 예시” 또는 “출력 기능 예정”으로 수정 |
| 3 | 회원 예시 `김라이트 (M/28)` | 가상 인물임을 더 명확히 할 필요 있음 | “가상 회원: 김라이트”로 수정 |

### 바로 교체할 문구

기존:
> QS: 지난 세션 대비 안정적 유지

수정:
> QS(내부 참고 점수): 지난 세션 대비 큰 변화 없음. 실제 판단은 트레이너 검토 후 확정합니다.

기존:
> 통증 증가 추세 확인

수정:
> 회원 자기보고 통증 점수가 소폭 증가했습니다. 트레이너가 운동 강도와 동작을 재확인합니다.

기존:
> 자세 관찰 신뢰도 낮음 (0.78)

수정:
> 촬영·관찰 신뢰도 0.78로 표시되어, 트레이너 재확인이 필요합니다.

### 추가 권장 고지

```html

본 화면은 LIGHT ONE 회원 리포트 화면 설계 시안이며, 표시된 이름·수치·점수는 모두 가상 예시입니다. QS·JATC는 내부 참고 지표로, 의료적 진단·치료·처방을 대체하지 않습니다. 최종 상담 내용은 트레이너 검토 후 확정됩니다.
```

## B. `lightone_infographic_v4.html`

### 현재 용도

촬영·분석·방법론 매칭 시스템 흐름도입니다. 일반 체형분석 솔루션과 LIGHT ONE의 차이를 설명하는 자료입니다.

### 종합 판정

내부 발표·멘토링 설명자료로 사용 가능. 다만 경쟁사 비교 표현과 기능 회복 프로토콜 표현은 완화해야 합니다.

### 유지할 부분

| 항목 | 이유 |
|---|---|
| “비의료 운동상담 보조 도구” 고지 | 규제 리스크를 줄이는 핵심 문구 |
| “표시값은 가상 예시” 고지 | 수치 오해를 줄임 |
| Human-in-the-loop 구조 | AI가 트레이너를 대체하지 않는 구조를 명확히 함 |
| 촬영 → 측정 → 점수화 → 변화 추적 → 트레이너 검토 흐름 | 서비스 차별성을 직관적으로 설명함 |

### 수정해야 할 사항

| 우선순위 | 현재 표현/영역 | 문제점 | 수정 권장안 |
|---:|---|---|---|
| 1 | “경쟁 솔루션이 멈추는 지점에서 시작합니다” | 경쟁사 기능 부재를 단정함 | “현재 조사 범위에서 많은 솔루션이 측정·리포트 중심으로 포지셔닝되어 있습니다” |
| 1 | “일반 체형분석 솔루션: 리포트 출력(종료)” | 모든 체형분석 솔루션이 리포트에서 끝난다고 단정 | “일반적인 측정형 솔루션 예시”로 변경 |
| 1 | “기능 회복 프로토콜 매칭” | 치료·재활로 오해될 수 있음 | “운동 상담 프로토콜 후보 매칭” 또는 “운동 시퀀스 후보 매칭” |
| 2 | “약화 근육군 회복 순서” | 의료·재활 판단처럼 보일 수 있음 | “관찰된 움직임 패턴에 따른 운동 우선순위 후보” |
| 2 | “회원 상담·재등록 설계” | 지나치게 영업 중심으로 보일 수 있음 | “회원 상담 자료 및 장기 관리 계획” |
| 3 | 일부 CSS `rgba(22,40,79,045)` 등 | alpha 값 표기가 잘못되어 렌더링 오류 가능 | `rgba(22,40,79,.045)`처럼 소수점 표기로 수정 |

### 바로 교체할 문구

기존:
> 측정 결과를 약화 근육군 회복 순서로 연결하고, 트레이너가 최종 확인합니다 — 경쟁 솔루션이 멈추는 지점에서 시작합니다.

수정:
> 측정 결과를 운동 상담 프로토콜 후보와 연결하고, 트레이너가 최종 확인합니다. 현재 조사 범위에서 많은 측정형 솔루션이 리포트 중심으로 제공되는 반면, LIGHT ONE은 상담 실행 단계까지 연결하는 구조를 지향합니다.

기존:
> 기능 회복 프로토콜 매칭

수정:
> 운동 상담 프로토콜 후보 매칭

기존:
> 회원 상담 · 재등록 설계

수정:
> 회원 상담 · 장기 관리 계획

### 기술 수정 체크리스트

- [ ] CSS의 `rgba(...,045)`, `rgba(...,07)` 형태를 `.045`, `.07`로 수정
- [ ] 경쟁 비교 표 제목을 “일반 체형분석 솔루션”에서 “일반 측정형 솔루션 예시”로 수정

# 3. 공통 수정 체크리스트

## 3-1. 표현 안전성

- [ ] `진단`, `치료`, `처방`, `재활`, `위험도 판정` 표현을 최소화
- [ ] `회복 프로토콜` → `운동 상담 프로토콜 후보`로 완화
- [ ] `통증 선별` → `통증 반응 기록`으로 완화
- [ ] `중단 권고` → `트레이너 검토 및 전문가 상담 안내`로 완화
- [ ] `경쟁사는 ~하지 못한다` → `현재 조사 범위에서는 ~중심으로 확인됨`으로 완화

## 3-2. 근거·출처

- [ ] RPE 출처: CDC 또는 원 논문 기준으로 정리
- [ ] NRS 출처: PubMed/임상척도 원문 기준으로 정리
- [ ] 비의료 건강관리서비스 가이드라인: 보건복지부 공식 PDF 확보
- [ ] 체력단련장업 수: 공공데이터포털 기준 최신 집계 재산정
- [ ] 도수치료 정책: 보건복지부 고시/보도자료 확인 전 사용 금지
- [ ] 경쟁사 고객 수: 각 회사 공식 홈페이지·IR·보도자료 원문으로만 사용
- [ ] 가격표: 경쟁 SaaS 가격 비교표 또는 고객 인터뷰 근거 확보

## 3-3. 기술·HTML

- [ ] `rgba(...,15)` → `rgba(...,.15)` 형태로 수정
- [ ] `Visual Rev.05`와 `Rev.6` 혼재 제거
- [ ] 외부 CDN 폰트 의존 시 PDF 변환본 별도 저장
- [ ] 이미지 base64가 큰 파일은 배포용에서 이미지 파일 분리 검토
- [ ] 실제 버튼이 아닌 경우 `PDF 다운로드 예시`, `상세 보기 예시`로 표기
