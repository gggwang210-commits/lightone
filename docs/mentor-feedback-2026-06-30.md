# LIGHT ONE 모두의 창업 멘토 피드백 분석

- 작성일: 2026-06-30
- 멘토: 황병선 / Bigbang Angels 대표
- 대상: LIGHT ONE / PT 트레이너 상담 리포트 SaaS

## 1. 원문 요약

멘토는 LIGHT ONE의 문제 인식은 현실적이라고 평가했다. 헬스장 회원 데이터가 분산되어 상담과 재등록 유도가 비효율적이라는 문제 정의는 타당하다. 다만 1기 심사에서 선정되지 못한 이유는 아이디어 자체보다 실행 가능성 검증 부족에 가깝다.

핵심 피드백은 다음과 같다.

1. **기술 역량 우려**  
   Python, 머신러닝, 딥러닝, DB, 웹 구현을 5개월 학습한 상태에서 B2B SaaS형 AI 모델 전체를 단독 구현하는 것은 일정·품질·보안 측면에서 불확실성이 크다. 기술 파트너 또는 공동창업자 확보가 우선이다.

2. **수익모델 검증 부족**  
   PT샵·필라테스 스튜디오는 소규모이고 마진이 두껍지 않다. 월 구독형 SaaS에 실제로 지불할 의향이 있는지 인터뷰 또는 사전 수요조사로 확인해야 한다.

3. **서비스 정의 과다**  
   초기부터 `AI SaaS`로 설명하기보다 `PT 트레이너를 위한 전용 리포트 도구`처럼 단순하고 직관적인 포지셔닝이 필요하다.

## 2. 전략적 해석

멘토의 조언은 아이디어 폐기가 아니라 **범위 축소 + 검증 우선 + 실행 리스크 제거**를 요구하는 피드백이다.

| 구분 | 현재 약점 | 2기 보완 방향 |
|---|---|---|
| 기술 | 단독 AI SaaS 구현 부담 | 기술 파트너 후보, 구현 범위 축소, 비의료 Human-in-the-loop 구조 명시 |
| 시장 | PT샵 지불의향 미검증 | 10~20명 수준의 트레이너 사전 수요조사, 구매의향서 또는 사용의향 확보 |
| 제품 | 기능 범위가 넓음 | 회원 1명의 변화 요약 리포트 MVP로 축소 |
| BM | B2B SaaS 단일 가정 | 트레이너 개인 결제형 B2C/B2Pro 초기 모델 병행 검토 |

## 3. 2기용 포지셔닝

> LIGHT ONE은 의료 진단 AI가 아니라, PT 트레이너가 회원의 체형·통증·운동목표·생활습관 상담 데이터를 구조화하고 회원 변화 리포트를 자동 생성해 상담·재등록·운동처방 설명을 보조하는 비의료 Human-in-the-loop 리포트 도구다.

초기 제품 설명은 다음 표현을 우선한다.

- PT 회원 변화 리포트 자동 생성 도구
- 트레이너 상담·재등록 보조 리포트
- 회원 상담 데이터를 기록 가능한 자산으로 바꾸는 PT 리포트 시스템

## 4. 30일 실행계획

### 1주차: 심사 리스크 제거

- 사업 설명에서 `AI 체형진단`, `통증 진단`, `의료 예측` 표현 제거
- `비의료`, `트레이너 보조`, `전문가 최종 판단`, `상담 리포트`로 재정의
- MVP 범위 1개로 축소: 회원 1명의 전후 변화 요약 리포트
- 기술 파트너 후보군 5명 리스트업

### 2주차: 수요검증

- PT 트레이너 10명 대상 사전 설문
- 질문 핵심: 월 구독 지불의향, 현재 상담기록 방식, 재등록 상담 어려움, 리포트 샘플 선호도
- 사용의향 3명 이상 또는 파일럿 참여의향 2곳 이상 확보 목표
- 가격 가정은 확정하지 말고 `지불의향 범위`로 수집

### 3주차: 리포트 MVP 제작

- 입력: 체형 사진, 기본 문진, 통증 부위, 운동목표, 생활습관, 트레이너 메모
- 출력: 회원 상태 요약, 변화 포인트, 상담 멘트, 다음 회차 관리 포인트
- 의료적 판단 문구 금지
- 샘플 리포트 3종 제작: 신규상담형, 재등록상담형, 변화관리형

### 4주차: 2기 제출 패키지 정리

- 문제정의 1페이지
- 고객검증 결과 1페이지
- MVP 샘플 3개
- 기술구현 범위와 파트너 확보 계획
- 개인정보·보안·비의료 경계 문서
- 2기 발표용 5분 스크립트
